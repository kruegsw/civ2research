## Data Interpretation Notes

The following fields from the save file have uncertain or partially-understood interpretations:

| Field | Offset | Raw Value | Uncertainty |
|-------|--------|-----------|-------------|
| `ratesRaw` (tax/science/luxury) | Per-civ +20 | Americans: 3, Celts: 5, Zulus: 4, Vikings: 4 | Single-byte encoding of three rates. Exact encoding not fully documented. Likely: rates are multiples of 10%, stored as a single value. Needs verification. |
| `simplifiedCombat` | 0x000C bit 4 | true | Flag 0x000C has dual purpose (file header + toggle flags). The bit layout overlaps with map height. Value `63` at 0x000C may not actually mean simplified combat is on — could be map dimension data. |
| `Settlers/Engineers count = 255` | Per-civ +214 | 255 for all civs | Every civ (including dead ones) shows 255 Settlers and 255 Engineers. This is almost certainly NOT a real count. Likely an initialization artifact or a counter that means something different for these unit types (e.g., work turns accumulated). |
| `reputation` | Per-civ +30 | Americans: 16, others: 0 | The reputation scoring scale is not fully documented. 16 could mean "slight reputation damage" or could be an index into a reputation category. |
| `techCount` field | Per-civ +16 | Stored as count+1 | The binary stores tech count plus one. Barbarians show -1 (stored as 0), which is correct (0 techs). Dead civs also show -1 despite having techs — possibly cleared on elimination. |
| `turnsForYear` vs `turnsPassed` | 0x001E / 0x001C | 680 / 135 | `turnsForYear` (680) is much larger than `turnsPassed` (135). The year calculation uses standard Civ2 progression (20yr/turn early). The 680 value's purpose and how it relates to the displayed year is unclear — may involve internal calendar mechanics. |
| Wonder `cityId` | 0x010A+ | e.g., 4 for Pyramids | These appear to be city **array indices** (0-based position in the city list), not sequence IDs. We map them by position in the parsed city array, which seems to produce correct results (Pyramids in Cardiff, etc.), but the binary analysis doc says these should be "city sequence IDs." The discrepancy needs verification with other save files. |
| City `buildings` bitmask | City +52 | e.g., 274 for Cardiff | The full building bitmask has ~30+ possible buildings. We only decode Palace (bit 1) and City Walls (bit 8). Other set bits represent buildings like Granary, Temple, Marketplace, etc. that are not decoded in this summary. |
| Barbarian treaty with Celts | Slot 0 treaties | Ceasefire | Barbarians having a ceasefire with Celts is unusual. May be an artifact of the treaty byte layout or a game mechanic specific to Deity difficulty. |
