## Function 1: FUN_0061a000 — `blit_sprite_16bit`

- **Address**: 0x0061A000 | **Size**: 326 bytes (large)
- **Category**: RN.sprite
- **Porting status**: SK
- **Proposed name**: `blit_sprite_16bit`

### Description

Rectangular sprite copy with transparency (color-key) and coordinate wrapping. Reads pixels from a source bitmap indexed through an FS-segment array, conditionally writes to a destination bitmap. Skips pixels matching the transparent color. Uses modulus for horizontal coordinate wrapping (tile wrapping for map sprites).

### Parameters (stack-based, 16-bit calling convention)

| Stack Offset | Type | Name | Description |
|-------------|------|------|-------------|
| +0x16 | far ptr | srcBitmap | Source bitmap base (segment:offset via FS lookup) |
| +0x1a | far ptr | dstBitmap | Destination bitmap base (segment:offset via FS lookup) |
| +0x22 | uint16 | srcStride | Source row stride (bytes per row in source) |
| +0x24 | uint16 | width | Number of columns to copy per row |
| +0x26 | uint16 | wrapModulus | Modulus for horizontal wrapping (e.g., map width) |
| +0x28 | uint16 | height | Number of rows to copy |
| +0x2a | uint16 | dstYOffset | Y-offset into destination index table |
| +0x2c | uint16 | srcYOffset | Y-offset into source index table |
| +0x2e | uint16 | dstXOffset | X-offset into destination index table |
| +0x30 | uint16 | srcXOffset | X-offset into source index table |
| +0x32 | int16 | transparentColor | Color key: pixels matching this value are skipped. If negative, no transparency check (all pixels copied) |

### Pseudocode

```
function blit_sprite_16bit():
    // Load source and destination bitmap far pointers from stack
    srcBase = stack[+0x16]    // far pointer (segment:offset)
    dstBase = stack[+0x1a]    // far pointer (segment:offset)

    rowsRemaining = stack[+0x28]    // height

    do:
        colsRemaining = stack[+0x24]    // width

        do:
            // Compute source lookup index
            // srcRowIndex = stack[+0x22] * (height - rowsRemaining)
            // srcColIndex = (width - colsRemaining)
            rowDelta = stack[+0x28] - rowsRemaining
            colDelta = stack[+0x24] - colsRemaining

            // Read source scanline pointer from index table
            // Uses a lookup table: srcLinePtr = FS:[srcBase + (srcYOffset + rowDelta) * 4]
            srcLineAddr = FS_LOOKUP[srcBase + (stack[+0x2c] + rowDelta) * 4]

            // Read source pixel with wrapping:
            // effectiveCol = (colDelta + srcLinePtr[colDelta]) % wrapModulus
            wrapIndex = (srcStride_entry + colDelta) % stack[+0x26]
            srcPixel = *(srcLineAddr + stack[+0x2a] + wrapIndex)

            // Transparency test
            transparentColor = stack[+0x32]
            if transparentColor < 0 OR srcPixel != (byte)transparentColor:
                // Write pixel to destination
                dstLineAddr = FS_LOOKUP[dstBase + (stack[+0x30] + rowDelta) * 4]
                *(dstLineAddr + stack[+0x2e] + colDelta) = srcPixel

            colsRemaining -= 1
        while colsRemaining > 0

        rowsRemaining -= 1
    while rowsRemaining > 0
```

### Key format details

- **FS-segment indexed bitmap access**: Both source and destination bitmaps are accessed via an indirection table indexed through FS segment. Pattern: `FS:[base + row * 4]` gives the address of scanline `row`. This is a classic DIB scanline pointer table.
- **Transparency**: `transparentColor` at stack +0x32 is a signed int16. If negative, all pixels are copied (no transparency). Otherwise, pixels equal to `(byte)transparentColor` are skipped.
- **Wrapping**: Column index is taken modulo `wrapModulus` (stack +0x26). This provides horizontal map wrapping for tile blitting on cylindrical maps.
- **No clipping**: The simple blit has no bounds clipping — caller must ensure the destination rect fits.

## Function 2: FUN_0061a759 — `blit_rle_sprite_16bit`

- **Address**: 0x0061A759 | **Size**: 423 bytes (large)
- **Category**: RN.sprite
- **Porting status**: SK
- **Proposed name**: `blit_rle_sprite_16bit` (already in rename_map.json)

### Description

RLE (run-length encoded) sprite blit with transparency, vertical flip, and horizontal clipping. Reads compressed scanline data from source, clips against destination bounds, and copies pixels with color-key filtering. Supports bottom-up bitmaps via negative stride (vertical flip). More complex than `blit_sprite_16bit` — used for sprites with large transparent regions where RLE compression saves bandwidth.

### RLE Scanline Format

Each RLE scanline record is structured as:

```
Offset -4: uint16 startX      // starting X position of non-transparent run
Offset -2: uint16 dataLen     // length in bytes of pixel data for this run
Offset  0: byte[dataLen]      // raw pixel data for the non-transparent run
```

Records are navigated by: `nextRecord = currentAddr + dataLen + 4` (skip past the pixel data + the 4-byte header of the next record).

### Pseudocode

```
function blit_rle_sprite_16bit():
    // Load parameters
    srcData       = stack[+0x0a]    // far ptr to RLE source
    dstBitmap     = stack[+0x0e]    // far ptr to destination
    dstStride     = stack[+0x20]    // signed: negative = bottom-up
    transparentC  = stack[+0x1a]    // color key byte

    // Handle vertical flip for bottom-up bitmaps
    if dstStride < 0:
        dstStride = -dstStride
        startRow = (stack[+0x22] - 1) - stack[+0x1e]   // mirror row
    else:
        startRow = stack[+0x1e]

    // Compute destination row pointer
    dstRowPtr = dstBitmap + startRow * dstStride + stack[+0x1c]

    // Set up source scanline navigation
    dstLookupPtr = (stack[+0x2a] + stack[+0x32]) * 2 + stack[+0x12]  // lookup table entry
    rlePtr = srcData + 4   // skip first 4 bytes (header)

    rowsLeft = stack[+0x26]   // numRows

    do:
        // ── Skip source scanlines until we reach the current destination row ──
        // Navigate RLE records: each record is accessed as:
        //   startX  = *(rlePtr - 4) as uint16
        //   dataLen = *(rlePtr - 2) as uint16
        //   nextRecord = rlePtr + dataLen + 4
        currentSrcRow = initialSrcRow
        while currentSrcRow < targetRow:
            dataLen = *(uint16*)(rlePtr - 2)
            rlePtr = rlePtr + dataLen + 4    // advance to next scanline
            currentSrcRow += 1

        // ── Read this scanline's RLE header ──
        startX  = *(uint16*)(rlePtr - 4)
        dataLen = *(uint16*)(rlePtr - 2)

        if dataLen != 0:
            // ── Horizontal clipping (left) ──
            // Compute destination write position
            dstWriteBase = (stack[+0x28] + stack[+0x30]) * 2 + stack[+0x08]
            savedDstRowPtr = dstRowPtr
            srcPixelPtr = startX   // offset into decompressed run

            // Skip source pixels that fall before the clip region
            clipLeft = stack[+0x24]   // clipWidth countdown
            while lookupTable[dstWriteBase] < startX:
                dstWriteBase += 2
                dstRowPtr += 1       // advance destination
                clipLeft -= 1

            // ── Copy pixel run with transparency ──
            if clipLeft > 0:
                pixelsLeft = clipLeft
                do:
                    pixelOffset = lookupTable[dstWriteBase] - startX
                    if pixelOffset >= dataLen:
                        break                    // past end of this run

                    srcPixel = rlePixelData[pixelOffset]
                    if srcPixel != transparentC:
                        *dstRowPtr = srcPixel    // write non-transparent pixel

                    dstWriteBase += 2
                    dstRowPtr += 1
                    pixelsLeft -= 1
                while pixelsLeft != 0

            dstRowPtr = savedDstRowPtr   // restore (we wrote into a copy)

        // Advance destination to next row
        dstRowPtr += dstStride           // may be negative (already negated above)

        // Advance source lookup pointer
        dstLookupPtr += 2

        rowsLeft -= 1
        if rowsLeft < 1:
            return
    loop forever
```

### Key format details

- **RLE record structure**: 4-byte header (startX:u16, dataLen:u16) followed by `dataLen` raw pixel bytes. This is NOT traditional byte-level RLE (skip/fill/copy commands) — it is a "span" format where each scanline stores a single non-transparent horizontal run with its starting X coordinate and raw pixel data.
- **Scanline navigation**: To reach scanline N, you must walk forward from the beginning: `ptr += *(ptr-2) + 4` for each scanline. This is a linked-list-like traversal, not random access.
- **Vertical flip**: When `dstStride < 0`, the function negates it and starts at `(height-1) - startRow` instead. This supports Windows bottom-up DIBs (the default for BMP/DIB surfaces).
- **Horizontal clipping**: Before copying pixels, the function skips source pixels whose X coordinate falls outside the destination column range. This allows blitting a sprite that is partially off-screen.
- **Transparency**: Each pixel is compared to `transparentColor`; matching pixels are skipped (destination unchanged). This is the color-key mechanism.
- **16-bit segment addressing**: All pointer arithmetic uses segment:offset pairs (in_SS, in_DS, FS_OFFSET). The FS segment points to a scanline lookup table (array of 32-bit pointers to scanline start addresses).
- **Likely dead or thunked**: These 16-bit routines cannot be called directly from 32-bit Win32 code. They are either: (a) invoked via a flat thunk layer, (b) called from other 16-bit code elsewhere in the binary, or (c) dead code preserved from linking the original MicroProse 16-bit graphics library.
