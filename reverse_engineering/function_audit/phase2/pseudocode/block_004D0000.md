### Block Overview

Three major subsystems:
1. **Wonder Movie / Advance Animation Window** (004D007E-004D6F58) — Throne room animation, wonder movies, typewriter text, starfield particles
2. **Rules Editor / Improvement Editor** (004D8AF0-004DAA51) — Cheat-mode improvement/wonder editor with RULES.TXT I/O
3. **Multiplayer Diplomacy / Parley System** (004DB210-004DF10F) — Network packet building, description text generation, transaction execution for MP negotiations

The 12 "missing from catalog" functions (004D49D5-004D4A56) are destructor stubs for the `build_advance_scene` function.
The 13th uncataloged function is FUN_004d6f58 (noop).

function build_advance_scene():

    // === SECTION 1: Room Layer Definitions (7 layers) ===
    // Each layer: [type, threshold_category, x, y, w, h, sprite_id]
    // layer[0]: type=2, cat=6, pos=(0x1cf,0xb2), size=(0xaa,0x2b), sprite=0x33, id=0x204
    // layer[1]: type=2, cat=6, pos=(0x1d0,0x189), size=(0x68,0x3d), sprite=0x28, id=0x230

    // Indexed progression sprites for floor/wall sections
    // Each entry: [sprite_base, y, x, w, h, blank, blank, threshold_level]
    // Entries at indices 0-10, thresholds: 0,1,2,3,4,6,8,10,12,14,16

    // === SECTION 3: Wall Upgrade Array (8 entries) ===
    // Indexed by threshold levels 0x17-0x1e
    // Similar structure to floor, different sprite coordinates

    // === SECTION 4: Decoration Objects (~80 entries) ===
    // Each: [type, cat, x, y, src_x, src_y, w, h, ???, threshold]
    // type 0-5: different rendering categories
    // cat: which civ power category (1-8) to compare threshold against
    // Types checked via FUN_00596c08(civ_id, category)

    // === DIALOG CREATION ===
    // Create 11 room layer dialogs with sequential IDs:
    //   0x1b9, 0x1e0, 0x1c7, 0x1ea, 0x1d6, 0x1ba, 0x1e1, 0x1c8, 0x1e9, 0x1b5, 0x1a7

        if category_value < object.threshold:

    size = 0x20                           // header: 32 bytes
    side_size = calculate_offer_size(type, side=0)
