### Pseudocode

```
function ai_try_settle_unit():
    // Operates on the "current unit" identified by global DAT_00655afe
    call FUN_00484d52()   // pre-processing helper (context setup?)

    unitIdx = (int)DAT_00655afe
    unitAlive     = units[unitIdx].alive         // +0x08 from unit base (DAT_006560f8)
    unitOrders    = units[unitIdx].orders         // +0x0F (DAT_006560ff)
    unitType      = units[unitIdx].unitTypeId     // +0x06 (DAT_006560f6)
    unitDomain    = unitTypes[unitType].domain     // DAT_0064b1c9[type*0x14]
    unitRange     = unitTypes[unitType].range      // DAT_0064b1ca[type*0x14]
    unitX         = units[unitIdx].x              // +0x00 (DAT_006560f0)
    unitY         = units[unitIdx].y              // +0x02 (DAT_006560f2)
    unitOwner     = units[unitIdx].owner          // +0x07 (DAT_006560f7)

    // Guard: unit must be alive, have settle orders (0x0B), be land domain (0), not air range (2)
    if unitAlive == 0: return
    if unitOrders != 0x0B: return      // 0x0B = "Build City" / settle order
    if unitDomain != 0: return         // must be land unit
    if unitRange == 2:
        // Range-2 = air unit; but also check if unit can reach its destination
        canReach = FUN_005b49cf(unitX, unitY, unitOwner)
        if canReach != 0: return       // can reach → don't override with settle

    // EITHER the settle logic applies directly, OR fall through to unit_ai_master
    // Try unit_ai_master first
    result = FUN_00538a29()            // unit_ai_master — main AI decision
    if result == 0:                    // unit_ai_master did NOT handle the unit
        stillAlive = FUN_005b2c3d(unitIdx)
        if stillAlive == 0: return     // unit died

    // Check scenario restriction
    restricted = FUN_004c5408(unitIdx)
    if restricted != 0: return         // scenario blocks this action

    // Execute: put unit to sleep or settle
    FUN_005b6787(unitIdx)
```

### Key references
- **Unit array base**: DAT_006560f0 (stride 0x20 = 32 bytes per unit record)
- **Unit type array base**: DAT_0064b1b8 (stride 0x14 = 20 bytes per type record)
- Domain offset +0x11 within type, Range offset +0x12 within type
- Orders 0x0B = "Build City" (settler auto-settle command)

### Missing from JS
- The `generateSettlerActions` in `cityai.js` handles city-site scoring but does NOT have this exact "try settle" dispatch loop. The binary's settle logic is a tight inner loop called repeatedly from the automation driver; JS uses a different action-generation architecture.

### Pseudocode

```
function ai_process_unit_automation():
    civIdx = (int)DAT_00655b05          // current AI civ being processed

    // Special flag: check if this is an automation-specific scenario mode
    bAutoSettlerMode = false
    if (DAT_00655af0 & 0x80) != 0       // game flag: has automation enabled
       AND (DAT_0064bc60 & 0x8000) != 0  // scenario flag
       AND DAT_00655af8 == 1             // game state == 1
       AND DAT_0064bcba == civIdx:       // scenario-specific civ matches
        bAutoSettlerMode = true

    // Two-pass loop: pass 0 = land units, pass 1 = sea units
    for pass = 0 to 1:
        // Iterate all unit slots in reverse order
        slotIdx = DAT_00655b16 - 1       // total unit slots (descending)
        while slotIdx >= 0:
            // Multiplayer abort checks
            if DAT_00655b02 > 2:         // multiplayer mode (>2 players?)
                if DAT_006ad698 != 0: return   // MP abort flag 1
                if DAT_006ad685 != 0: return   // MP abort flag 2

            // Filter: unit must be alive AND owned by current civ
            unitSeqId = units[slotIdx].sequenceId   // +0x1A (DAT_0065610a)
            unitOwner = units[slotIdx].owner         // +0x07
            unitType  = units[slotIdx].unitTypeId    // +0x06
            typeDomain = unitTypes[unitType].domain   // +0x11 in type record

            if unitSeqId == 0: skip      // dead unit
            if unitOwner != civIdx: skip // not our unit

            // In auto-settler mode, pass 0 only processes sea-domain units (domain==1)
            if bAutoSettlerMode:
                if pass == 0 AND typeDomain != 1: skip  // pass 0: only sea

            iterCount = 0
            while FUN_005b6458(slotIdx) != 0:   // unit can still act (has moves)
                // Multiplayer sync
                if DAT_00655b02 > 2:
                    FUN_004b0b53(0xFF, 2, 0, 0, 0)   // send MP packet
                    XD_FlushSendBuffer(5000)           // flush with 5s timeout
                    FUN_0047e94e(1, 0)                 // process incoming msgs

                // Save current total unit count to detect unit creation/death
                savedUnitCount = DAT_00655b16
                DAT_00655afe = slotIdx               // set "current unit" global
                ai_try_settle_unit()                  // attempt settle/automation

                if DAT_00655b16 != savedUnitCount: break  // unit count changed → restart

                // If unit didn't settle and isn't in "settle orders" mode,
                // and its type is not sea-domain, increment iteration counter faster
                if units[slotIdx].orders != 0x0B
                   AND unitTypes[unitType].domain != 1:
                    iterCount += 1   // extra increment for non-settle, non-sea
                iterCount += 1

                // Safety valve: prevent infinite loops
                if iterCount > 20:
                    FUN_005b6787(slotIdx)   // force sleep
                    break

                // On pass 0, for non-barbarian civs, check if unit is on land
                // and could be a settler that needs to found a city
                if pass == 0 AND civIdx != 0:
                    domain = unitTypes[units[slotIdx].unitTypeId].domain
                    if domain != 0: continue  // skip non-land on pass 0

                    alive = FUN_005b2c3d(slotIdx)
                    if alive == 0:  // unit died
                        isLand = FUN_005b89e4(unitX, unitY)
                        if isLand != 0:  // tile is land
                            stackSize = FUN_005b50ad(slotIdx, 2)
                            if stackSize > 1:  // multiple units here
                                // Check territory or vision
                                hasTerritory = (DAT_00655b0b & units[slotIdx].visibility) != 0
                                if not hasTerritory:
                                    canReach = FUN_005b49cf(unitX, unitY, civIdx)
                                    if canReach == 0: continue
                                ai_try_settle_unit()  // retry settle

            FUN_0041033a()   // post-unit cleanup (redraw? state reset?)
            slotIdx -= 1
```

### Key references
- **DAT_00655b05**: current AI civ index (0-7)
- **DAT_00655afe**: current unit index being processed
- **DAT_00655b16**: total unit slot count
- **DAT_00655b02**: player count (>2 = multiplayer)
- **DAT_006ad698, DAT_006ad685**: MP abort flags
- **DAT_00655af0**: game flags byte (bit 0x80 = automation?)
- **DAT_0064bc60**: scenario flags (bit 0x8000)
- **DAT_0064bcba**: scenario civ assignment
- **DAT_00655b0b**: visibility bitmask for current civ

### Missing from JS
- JS `runAiTurn()` calls separate action generators (settler, military, production, etc.) then applies actions via reducer. This binary function is a direct imperative loop that mutates game state in-place. The two-pass (land then sea) architecture is not replicated in JS. The 20-iteration safety valve and multiplayer sync are not present.

- **Address**: 0x00548B70 | **Size**: 264 bytes

- **Called by**: `draw_icon_row_spaced` (0x548C78)

### Pseudocode

```
function calc_icon_spacing(count, iconSize, totalSpace, out_fittedCount, out_remainder) -> int:
    // Clamp minimums
    if iconSize < 2: iconSize = 1
    if count < 2: count = 1

    effectiveSize = iconSize

    if count == 1:
        if out_remainder != null: *out_remainder = 0
        if out_fittedCount != null: *out_fittedCount = 1
    else:
        availSpace = totalSpace - iconSize       // space minus one icon
        spacing = availSpace / (count - 1)       // even spacing between items
        if out_remainder != null: *out_remainder = availSpace % (count - 1)
        if out_fittedCount != null: *out_fittedCount = count

        if spacing < iconSize:                   // items overlap
            effectiveSize = spacing
            if spacing < 1:                      // can't even fit 1px per item
                if out_remainder != null: *out_remainder = 0
                if out_fittedCount != null: *out_fittedCount = (totalSpace - iconSize) + 1
                effectiveSize = 1

    return effectiveSize
```

- **Address**: 0x00548C78 | **Size**: 246 bytes

- **Porting status**: SK
- **Calls**: `calc_icon_spacing` (0x548B70), `FUN_005cef31` (blit_normal)

### Pseudocode

```
function draw_icon_row_spaced(sprite, surface, x, y, filled, total, iconSize, totalWidth, distribute):
    accumRemainder = 0
    origTotal = total

    // Calculate spacing
    iconSize = calc_icon_spacing(total, iconSize, totalWidth, &total, &remainder)

    // If count changed and filled != total, scale filled proportionally
    if origTotal != 0 AND total != origTotal AND filled != origTotal:
        filled = (total * filled) / origTotal

    // Draw each icon
    for i = 0; i < min(total, filled); i++:
        blit_normal(sprite, surface, x, y)   // FUN_005cef31
        x += iconSize

        // Sub-pixel distribution: spread remainder evenly
        if distribute AND total > 1:
            accumRemainder += remainder
            while accumRemainder >= (total - 1):
                accumRemainder -= (total - 1)
                x += 1
```

```
function save_events_file() -> int:
    backupFile = null
    outputFile = null
    filename = "EVENTS." + DAT_0062cd24    // e.g., "EVENTS.TXT"
    success = 0

    savedCwd = getcwd()
    chdir(DAT_0064bb08)     // scenario directory

    // Remove old backup, rename current to backup
    if file_exists("EVENTS.BAK") AND remove("EVENTS.BAK") != 0: goto cleanup
    if file_exists(filename) AND rename(filename, "EVENTS.BAK") != 0: goto cleanup

    backupFile = fopen("EVENTS.BAK", "r")
    outputFile = fopen(filename, "w")
    if outputFile == null: goto cleanup

    if backupFile == null:
        // No existing file — write fresh
        fputs("@BEGINEVENTS\n", outputFile)
    else:
        // Copy everything before @BEGINEVENTS from backup
        while true:
            line = fgets(buf, 256, backupFile)
            if line == null: goto cleanup

    write_all_events(outputFile)

        success = 1
    else:
        // Skip old events in backup until @ENDEVENTS

            line = fgets(buf, 256, backupFile)
            if line == null: goto cleanup

            if strcmpi(trimmed, "@ENDEVENTS") == 0: break
        fputs("@ENDEVENTS\n", outputFile)
        // Copy everything after @ENDEVENTS from backup
        while true:
            line = fgets(buf, 256, backupFile)
            if line == null: { success = 1; break }
            if fputs(buf, outputFile) == -1: break

cleanup:
    if backupFile != null: fclose(backupFile)
    if outputFile != null: fclose(outputFile)
    chdir(savedCwd)
    return success
```

### Pseudocode

```
function get_nth_set_bit(n, bitmask) -> uint:
    // Returns the bit value (power of 2) of the Nth set bit in bitmask
    // n is 0-based index into set bits
    bit = 0
    do:
        if bit == 0: bit = 1
        else: bit <<= 1

        // Skip unset bits
        while (bitmask & bit) == 0:
            bit <<= 1

        wasNonZero = (n != 0)
        n -= 1
    while wasNonZero

    return bit
```

### What's Missing from JS Port (AI functions only)

1. **Two-pass land/sea automation loop** (`ai_process_unit_automation`): JS `runAiTurn` does not separate passes by domain. All unit processing is mixed.

2. **20-iteration safety valve**: Binary forces units to sleep after 20 failed action attempts. JS has no equivalent deadlock prevention.

3. **Multiplayer sync points**: Binary calls `XD_FlushSendBuffer` and processes network messages between individual unit moves. JS processes the entire AI turn atomically.

4. **Auto-settler scenario mode**: The `bAutoSettlerMode` flag logic (checking game flags 0x80, scenario flag 0x8000, and civ assignment) has no JS equivalent.

5. **Direct settle dispatch**: `ai_try_settle_unit` checks orders==0x0B and directly dispatches to `FUN_005b6787` (execute settle). JS generates action objects that go through the reducer.
