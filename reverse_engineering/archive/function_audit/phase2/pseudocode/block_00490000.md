## 3. AI.GoalManagement — AI Goal/Threat Priority Lists

Two per-civ priority-sorted lists within the civ struct (stride 0x594):
- **List A** (civ+0x414, offset DAT_0064cab4): 48 entries, 6 bytes each `[x:int16, y:int16, type:int8, priority:int8]`
- **List B** (civ+0x534, offset DAT_0064cbd4): 16 entries, same format

### FUN_00492b60 — ai_negate_goal_priority
- **Category**: AI.GoalManagement
- **Size**: 181 bytes
- **Ported**: R (referenced in unitai.js line 3096 context)
- **Refs**: civ+0x419 (list A priority byte)

**Pseudocode**:
```
function ai_negate_goal_priority(civ_id, slot_idx):
    priority = listA[civ_id][slot_idx].priority
    absPriority = abs(priority)
    listA[civ_id][slot_idx].priority = -absPriority
    // Effectively marks goal as "cancelled" (negative = inactive)
```

**Key insight**: Absolute value taken first, then negated. This ensures already-negative priorities become more negative (double-negate = still negative). This is the "decay" step for AI goal aging.

### FUN_0049301b — ai_add_goal_a

    // Phase 1: Check if exact match already exists with equal-or-higher priority
    for i = 0 to 47:
        if listA[civ_id][i] matches (x, y, type):
            if listA[civ_id][i].priority >= priority:
                return  // already have equal or better goal here
            break

        // Only for human civ AND type 2 or 3 (attack/defend goals)
        if civ_id == human_civ AND NOT AI_controlled AND (type == 2 OR type == 3):

                    goalContinent = get_continent(x, y)

                        maxMoves = get_max_moves(unit)

                            unit.orders = 0x0B  // GOTO

                            unit.gotoY = y

### FUN_00493602 — ai_decay_and_merge_goals
- **Category**: AI.GoalManagement
- **Size**: 365 bytes
- **Ported**: TM
- **Refs**: ai_negate_goal_priority, ai_add_goal_a

**Pseudocode**:
```
function ai_decay_and_merge_goals(civ_id):
    // Step 1: Decay list A — negate priorities of negative entries, clear dead ones
    for i = 0 to 47:
        if listA[civ_id][i].priority < 0:
            listA[civ_id][i].type = 0xFF  // remove decayed entries
        ai_negate_goal_priority(civ_id, i)  // flip sign

    // Step 2: Merge list B into list A
    for i = 0 to 15:
        if listB[civ_id][i].type >= 0:  // not 0xFF
            ai_add_goal_a(civ_id,
                listB[civ_id][i].x,
                listB[civ_id][i].y,
                listB[civ_id][i].type,
                listB[civ_id][i].priority)
```

**Key insight**: Called once per turn. Goals that were negated last turn (priority < 0) get removed. Active goals get their priority negated (marking them for potential removal next turn unless refreshed). Then short-term goals from List B are promoted to List A. This creates a 2-turn decay cycle: goals not refreshed within 2 turns are automatically removed.
