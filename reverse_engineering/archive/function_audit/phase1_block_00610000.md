## SUMMARY

Block 00610000 contains exactly **2 functions**, both large (326 and 423 bytes respectively). They are **16-bit legacy sprite blitting routines** using segmented x86 memory addressing, which is inconsistent with the rest of the Win32 codebase. This strongly suggests they originate from an older MicroProse 16-bit graphics library (likely DOS or Win16 era) that was linked into the Win32 binary -- possibly via a flat thunk or as pre-compiled 16-bit object code embedded in the executable.

**FUN_0061a000** is a simple rectangular sprite copy with transparency (color-key) support and coordinate wrapping (modulus on column index). **FUN_0061a759** is an RLE-compressed sprite blit with transparency, vertical flip support (negative stride), and horizontal clipping. Both operate on raw pixel buffers accessed through segment:offset pairs.

Neither function references any known Civ2 game-state globals (DAT_ addresses), which is consistent with them being low-level graphics primitives that predate the Win32 game engine. They are likely called indirectly (via function pointers or thunks) from the 32-bit sprite pipeline documented in the reference context (e.g., the `pixel_copy`/`pixel_fill` family at 005E518E/005E52BF), or they may be entirely dead code preserved from linking.
